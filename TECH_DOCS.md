# Техническая документация: Архитектура и Торговая логика LTC

## 1. Введение в архитектуру (Liquid Time-Constant)
Обычные рекуррентные нейросети (RNN, LSTM, GRU) работают в дискретном времени и требуют жестко заданных временных шагов.
В отличие от них, **Liquid Time-Constant (LTC)** сети:
- Построены на дифференциальных уравнениях, что позволяет им работать в **непрерывном времени**.
- Способны менять свою «постоянную времени» в зависимости от входящих данных (отсюда название "жидкие").
- Имеют крайне малый размер (от 16 до 64 нейронов), но при этом превосходят классические модели по способности улавливать паттерны из зашумленных данных.

Для реализации в данном проекте используется библиотека `ncps` (Neural Circuit Policies) от MIT. Конфигурация синапсов генерируется автоматически алгоритмом `AutoNCP`.

## 2. Пайплайн обработки данных

### 2.1. Получение данных (WebSocket & REST API)
- Данные о сделках поступают в реальном времени через Binance WebSocket (`wss://stream.binance.com:9443/ws/btcusdt@trade`).
- WebSocket возвращает `p` (цена) и `q` (количество/объём) каждой сделки.
- Свечи формируются вручную: накапливаются цены и объёмы до закрытия таймфрейма.
- Для скрипта `finance_bot_trader_minutes.py` реализован механизм "быстрого старта": скрипт через REST API (`https://api.binance.com/api/v3/klines`) запрашивает последние `SEQ_LENGTH` свечей, чтобы сразу инициализировать буфер памяти.

### 2.2. Нормализация (MinMaxScaler)
Нейросети крайне плохо работают с абсолютными числами (например, $70,000). Перед отправкой в нейросеть цена нормализуется с помощью `sklearn.preprocessing.MinMaxScaler`, превращаясь в значение от 0 до 1.
- **Цена:** диапазон 50,000 — 100,000 USD
- **Объём:** диапазон 0 — 1,000 BTC
- Вектор признаков: `[цена_normalized, объём_normalized]` → 2D тензор

### 2.3. Буферизация (Sequence Window)
Скрипт поддерживает "окно" из последних N значений (в коде `SEQ_LENGTH = 50`). При поступлении нового значения, самое старое удаляется. Это формирует тензор размерности `[1, 50, 2]` (batch, seq_len, features), который и подается на вход LTC.

## 3. Логика Online Learning (Обучения на лету)
Главная инновация скриптов — отказ от предварительного обучения (Pre-training).
Каждый раз, когда закрывается новая свеча (или поступает новый тик):
1. **Обучение:** Скрипт берет буфер из `[T-50 до T-1]` значений и пытается предсказать текущее известное значение `T`. Вычисляется ошибка (`MSELoss`) и происходит обновление весов (`optimizer.step()`).
2. **Прогноз:** Сразу после обучения, скрипт сдвигает окно на один шаг `[T-49 до T]` и просит нейросеть предсказать **следующее, еще не существующее значение `T+1`**.

## 4. Торговая стратегия (Threshold & Fees)
Предсказанное значение `T+1` сравнивается с текущей ценой `T`.
Для избежания убытков из-за микро-колебаний и комиссий биржи, введен порог реакции (`THRESHOLD`):
`THRESHOLD = FEE_USD + MIN_PROFIT`
- **BUY Signal:** Выдается, если `Прогноз - Текущая цена > THRESHOLD`. Ожидаем резкий рост.
- **SELL Signal:** Выдается, если `Прогноз - Текущая цена < -THRESHOLD`. Ожидаем резкое падение.
- **WAIT:** Во всех остальных случаях нейросеть рекомендует находиться вне позиции, так как предсказанное движение слишком мало и будет съедено комиссией.

## 5. Расширение проекта
Для полноценного трейдинга (Автоторговли) необходимо интегрировать библиотеку `python-binance`:
1. В секцию `if signal == 'BUY':` добавить вызов `client.create_order(side=SIDE_BUY, type=ORDER_TYPE_MARKET)`.
2. Добавить логику риск-менеджмента: проверку текущего баланса и жесткий Stop-Loss (автоматическое закрытие позиции при неблагоприятном движении цены).
